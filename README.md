## A mobile application built with *Flutter and Dart* to help physiotherapists track and visualise patient progress over time.
A mobile app that helps physiotherapists track patient pain levels and 
visualise recovery trends over time — built to address a real gap I saw 
Firsthand: physiotherapy in Pakistan is still largely run on paper charts 
and handwritten notebooks, making it hard to track patient progress 
systematically or make evidence-based treatment decisions.

## My Role
I served as the **Clinical Subject Matter Expert (SME)** — defining what 
data actually matters in a physiotherapy context (pain scale, entry 
structure, progress tracking logic) and validating the app's workflow 
against real clinical practice. Built in collaboration with Mohib Abdali, 
who handled development.

## 📌 Features

**Patient Management**
- Add patient details: Name, Age, Gender, and Diagnosis
- Track multiple progress entries per patient

**Progress Tracking**
Each entry includes:
- Pain Level (0–10 scale)
- Description / Notes
- Date of Entry

**Data Visualisation**
- Line graph showing pain level trends over time, supporting quick 
  identification of improvement or regression

## 🛠️ Tech Stack
- **Flutter** – Cross-platform UI framework
- **Dart** – Application logic

## 🚀 How It Works
1. Add a new patient with their details
2. Log progress entries (pain level, notes, date)
3. Build a timeline of patient recovery
4. View a line chart to analyse trends across sessions

## 🎯 Use Case
Designed for physiotherapists who want a fast, structured way to track 
recovery patterns and support evidence-based treatment decisions — 
replacing the handwritten notebooks still common in clinical practice.

## ⚠️ Note on Data
This is a prototype built and tested with sample/mock patient data only — 
no real patient information was used or stored.
